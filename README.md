# algorithms

### 🗂️ 폴더명 규칙

- **`[플랫폼_이니셜]_[문제_번호]_[문제_이름]`** 형식으로 폴더명을 명명합니다.

  - **플랫폼 이니셜 예시:**
    - `BOJ`: 백준 온라인 저지 (Baekjoon Online Judge)
    - `PG`: 프로그래머스 (Programmers)
    - `LC`: 리트코드 (LeetCode)
    - `SWEA`: SW Expert Academy
    - `ETC`: 기타 플랫폼 또는 특정 플랫폼에 속하지 않는 문제
  - **문제 번호:** 해당 플랫폼의 문제 고유 번호 (예: 백준 1000번, 프로그래머스 12345번)

### 📄 파일명 규칙

- **풀이 코드:**

  - `solution.[확장자]` 형식으로 저장합니다.
  - **확장자 예시:** `.py` (Python), `.js` (JavaScript), `.java` (Java), `.cpp` (C++)
  - **예시:** `solution.py`, `solution.js`

- **추가 문서 (선택 사항):**

  - `README.md`: 문제의 원문 설명이나 핵심 내용을 요약합니다.

    - 예시 템플릿

      ### [문제 번호] [문제 이름]

          ## 🔗 문제 링크
          [문제 링크](문제_URL_여기에_삽입)

          ## 📝 문제 설명
          (여기에 문제의 원문 또는 핵심 내용을 요약하여 붙여넣습니다.)
          ...

          ## ✨ 풀이 전략 / 핵심 아이디어
          (이 문제를 어떤 알고리즘/자료구조로 접근했는지, 왜 그렇게 생각했는지 등)
          ...

          ## ⏱️ 시간 복잡도 및 공간 복잡도
          (예: O(N), O(N log N) / O(N) 등)
          ...

          ## 💭 회고 / 어려웠던 점
          (문제를 풀면서 겪었던 시행착오, 새로 알게 된 점, 개선할 부분 등 개인적인 생각)
          ...
              * `test_cases.txt`: 직접 만들거나 추가로 테스트해본 입력 및 출력 예시를 기록합니다.

### 💡 Git Commit

- **커밋 메시지:**
  - `feat(PG_문제이름): 파이썬 풀이 추가`
  - `fix(BOJ_문제이름): 엣지 케이스 처리 로직 수정`
  - `refactor(LC_문제이름): 큐 대신 스택으로 풀이 변경`
- **브랜치:** (선택 사항) 복잡한 문제나 여러 시도를 할 경우, 별도의 브랜치를 생성하여 작업 후 `main` 브랜치에 병합합니다.
